## Design

Two explicit circuit scopes are now used:

- `uncontrolled_pf_one_step`
- `controlled_pf_time_evolution_block`

The uncontrolled scope remains the default for existing APIs and architecture sweep configs. A controlled block is selected explicitly and is defined as:

```text
C-U_PF(t_k),  t_k = 2^k t
```

`t` is the existing base step time from `surface_code_step_time`. `U_PF(t_k)` is generated by passing `t_k` once into the product-formula generator. It is not implemented as `[U_PF(t)]^(2^k)`.

The control qubit is appended at the last logical index. Therefore:

```text
num_logical_qubits = num_system_qubits + 1
control_qubit_index = num_system_qubits
```

The implementation first builds the system PF circuit at `effective_evolution_time`, converts that circuit to a one-control Qiskit gate, appends it with the last qubit as control, then decomposes to the existing `rz/cx/sx/x` QASM2 basis.

## Identity Phase

For controlled evolution, the identity coefficient is computed from the actual JW/grouped operator terms, not from the PySCF nuclear constant alone.

The phase accumulated by identity terms is computed using the same product-formula coefficient sequence as the non-identity terms:

```text
theta = -sum_j w_j * c_identity,j * t_k
```

This `theta` is emitted as a `PhaseGate` on the control qubit. The final circuit global phase may be normalized, but the control=0/control=1 relative phase remains a real gate. The uncontrolled path still drops the identity term as before.

Prepared artifact cache keys now include circuit scope, `qpe_power_k`, effective time, control qubit count, qret executable hash, and qret core library hash. Compile cache payloads also include the new circuit metadata. RZ helper independent cache payloads receive a scope identity so controlled/uncontrolled/k variants do not collide.

## Architecture Sweep

Architecture sweep accepts `circuit_scope` and `qpe_power_k` in `defaults` or per case.

Uncontrolled rows keep:

```text
qpe_scaling_model = linear_extrapolation_from_uncontrolled_pf_one_step
```

Controlled rows use:

```text
qpe_scaling_model = none_single_controlled_block
```

For controlled rows, QPE total fields such as `qpe_action_count`, `total_magic_state_count`, `total_runtime_*`, and `total_qubit_volume` are `None` with reason `single_controlled_block_not_qpe_total`.
